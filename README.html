⚽ Panini World Cup 2026 - Sticker Tracker App

📖 Overview

Panini World Cup 2026 is a modern, offline-first Android application designed to track, manage, and optimize sticker collections for the FIFA World Cup 2026 official Panini album. Built with cutting-edge Android development practices, it provides a seamless user experience for registering new stickers, tracking duplicates, executing exchanges, and retrieving real-time player information via external APIs.

✨ Key Features

Feature Description
📊 Real-time Dashboard Instant counters for Obtained, Pending, and Repeated stickers
🔍 Live Search Filter stickers by code, player name, or country in real-time
📥 Smart Registration Auto-detects new vs. duplicate stickers and updates counts
🔄 Exchange System Trade duplicates for missing stickers with inventory validation
🖼️ Panini-Style Player Cards Beautiful, FIFA-inspired UI cards with live player data
📡 API Integration Fetches real player photos, stats, and bios from TheSportsDB

🌙 Modern UI/UX Material 3, smooth animations, responsive layouts, and intuitive navigation

🏗️ Architecture & Enterprise Design

📐 Architectural Pattern

The app follows a Modern MVVM (Model-View-ViewModel) architecture enhanced with Layered Separation and Offline-First principles:

  1. UI Layer (Compose Screens + Navigation)
    → Observes StateFlow from ViewModel, renders declarative UI, handles user input.
  2. ViewModel Layer
    → Business logic coordinator. Transforms DAO flows into UI state, manages coroutines, handles validation.
  3. Data Layer (Room DAO + Network Module)
    → Single source of truth. Room handles local SQLite persistence; Retrofit handles external API calls.
  4. Domain/Model Layer
    → Plain Kotlin data classes (StickerEntity, PlayerDto) with clear boundaries between local and remote representations.

Enterprise Design Principles Applied:

  • ✅ Single Responsibility Principle (SRP): Each file/class has one clear purpose.
  • ✅ Separation of Concerns: UI, logic, and data are strictly isolated.
  • ✅ Reactive Programming: StateFlow + collectAsState() ensures UI always reflects current data.
  • ✅ Offline-First: App works 100% without internet. API calls are optional enhancements.
  • ✅ Scalable Structure: Ready for DI (Hilt), testing, and cloud sync without refactoring.

🔌 API Integration & Data Flow

🌐 External APIs Consumed

1. TheSportsDB API (Live Player Data)

  • Endpoint: GET https://www.thesportsdb.com/api/v1/json/3/searchplayers.php?p={playerName}
  • Purpose: Fetches real-time player photos, nationality, position, birth date, team, and bio.
  • JSON Mapping: PlayerResponseDto ↔ Kotlin data class via kotlinx.serialization
  • Error Handling: Graceful fallback UI if player isn't found or network fails.
  • Image Loading: Coil asynchronously loads strCutout or strThumb with placeholder & error states.

2. Render Backend API (Reference Architecture)

  • Base URL: https://contro-panini-api.onrender.com
  • Endpoints Documented:
    • GET /api/stickers/obtained → Collected stickers
    • GET /api/stickers/pending → Missing stickers
    • GET /api/stickers/repeated → Duplicates with counts
    • POST /api/stickers/register → Register new/duplicate
    • POST /api/stickers/exchange → Trade logic validation
  • Implementation Note: The app implements the exact same business logic locally via Room for 100% offline reliability. The API endpoints are fully documented and ready for future cloud sync implementation without changing UI/ViewModel layers.

🗄️ Local Database (SQLite via Room)

Aspect Implementation
Engine SQLite 3 (wrapped by Android Room)
Schema stickers table: code (PK), playerName, team, isObtained, repeatedCount
Reactive Queries Returns Flow<List<StickerEntity>> for automatic UI updates
Pre-population Seeds 100+ real players (ARG, BRA, POR, ESP, FRA, ALE, ING) on first launch
Thread Safety All write operations run on Dispatchers.IO via suspend functions
Data Integrity OnConflictStrategy.REPLACE prevents duplicate PK entries

🚀 Setup & Installation

📋 Prerequisites

  • Android Studio Iguana/Jellyfish/Koala+
  • JDK 17 or higher
  • Android SDK 34 (API 34)
  • Min SDK: API 26 (Android 8.0)

⚙️ Quick Start

  1. Clone the repository:
    git clone https://github.com/your-username/panini-worldcup-2026.git
  2. Open in Android Studio → Wait for Gradle sync
  3. Ensure gradle/libs.versions.toml and app/build.gradle.kts match the catalog structure
  4. Run on emulator or physical device:
    ./gradlew installDebug
  5. 🎉 App launches with pre-loaded album and offline functionality ready.
